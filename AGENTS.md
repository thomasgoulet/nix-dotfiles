# Repository

This repository lives at `~/.config` and serves two purposes simultaneously: it is a Nix flake (under `~/.config/flake/`) that declaratively manages the entire system, and it is a standard XDG config directory whose files are tracked in git. Some config files (for example `starship.toml`, `git/config`) are generated by Home Manager and symlinked here; others are plain files committed directly. The `.gitignore` uses an opt-in strategy; everything is ignored by default, with specific paths explicitly tracked. This prevents accidentally committing secrets or files generated by Nix.

# How to Rebuild

Rebuilds are driven by `nh` under the hood, with `NH_FLAKE` and `NH_HOST` set system-wide as environment variables so no flags are needed.

The `nushell` module `nushell/modules/nix.nu` provides the day-to-day interface. And should be used when changing the configuration. `nix diff` allows to validate

| Command             | Effect                                                                                                                                   |
| ------------------- | ---------------------------------------------------------------------------------------------------------------------------------------- |
| `nix diff`          | Show the package diff between current and next generation. Should be used whenever changes are being made to validate compilation first. |
| `nix switch`        | Build and activate. Equivalent to `nh os switch`.                                                                                        |
| `nix update`        | Update flake inputs (`nix flake update`) then switch.                                                                                    |
| `nix rollback`      | Roll back to the previous generation or a specific one.                                                                                  |
| `nix generations`   | List all generations with timestamps.                                                                                                    |
| `nix flake-repl`    | Open `nix repl` with the flake loaded.                                                                                                   |
| `nix install <pkg>` | Ad-hoc `nix-shell` with a package.                                                                                                       |

# Flake Structure

**Entrypoint:** `flake/flake.nix`

The flake uses [`flake-parts`](https://github.com/hercules-ci/flake-parts) for output composition and [`den`](https://github.com/denful/den) as an abstraction layer over NixOS and Home Manager.
`den` introduces **aspects** (role bundles) and **hosts** that compose in a declarative way.

## Inputs

| Input            | Pin               | Purpose                        |
| ---------------- | ----------------- | ------------------------------ |
| `nixpkgs`        | `nixos-unstable`  | Primary package set            |
| `nixpkgs-stable` | `25.11`           | Available but not heavily used |
| `nixos-wsl`      | `main`            | WSL2 integration module        |
| `flake-parts`    | latest            | Flake output composition       |
| `home-manager`   | follows `nixpkgs` | Home Manager, unstable channel |
| `den`            | latest            | Aspect/host/user abstraction   |

## Module Layout

```
flake/
├── flake.nix
├── flake.lock
└── modules/
    ├── nixos/ # Hosts
    ├── aspects/ # Shared den aspects to be re-used
    └── homes/ # Home-manager configurations
```

# Editor

Configuration lives in `modules/aspects/editor/`. The aspect entry point is `modules/aspects/shell.nix`.

| File                  | Purpose                                                 |
| --------------------- | ------------------------------------------------------- |
| `helix-settings.nix`  | Editor settings (theme, shell, keybindings, UI options) |
| `helix-languages.nix` | Per-language LSP and formatter wiring                   |
| `packages.nix`        | Nix packages for all LSPs and formatters                |
| `broot.nix`           | `broot` file manager configuration                      |
| `lazygit.nix`         | `lazygit` configuration, wired to `helix` and `zellij`  |

# Conventions & Patterns

## `den` batteries

`den.batteries.inputs'` and `den.batteries.self'` are enabled on the host, making `inputs'` and `self'` available in every module without manual threading. This is how Home Manager modules access flake-local and upstream-flake packages.

## Theme

Every tool uses the same Catppuccin Mocha theme.
